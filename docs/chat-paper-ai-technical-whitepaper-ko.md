# Chat Paper AI 기술백서

**버전:** 1.0  
**프로젝트:** Chat Paper AI  
**저장소:** <https://github.com/hyeonjo00/chat-paper-platform>  
**분야:** 한국어 우선 AI SaaS, 대화 분석, 학술 논문 생성

## 초록

Chat Paper AI는 카카오톡 대화 내보내기 파일과 AI 대화 로그를 학술 논문 형식의 문서로 변환하는 한국어 우선 AI SaaS 플랫폼이다. 시스템은 파일 업로드, 대화 파싱, 익명화, 언어 감지, 청크 기반 AI 분석, 논문 생성, 결과 대시보드와 논문 리더를 하나의 흐름으로 연결한다.

제품의 핵심 방향은 사용자가 회원가입 없이 바로 업로드하고 결과를 확인할 수 있는 낮은 진입장벽이다. 원본 업로드 파일은 영구 저장하지 않고, 논문 생성과 결과 확인에 필요한 파싱 및 익명화된 대화 데이터만 데이터베이스에 저장한다.

## 1. 해결하려는 문제

대화 데이터에는 주제 변화, 감정 흐름, 관계 역학, 발화자 역할, 커뮤니케이션 습관처럼 사람이 직접 읽어도 구조화하기 어려운 정보가 포함된다. 일반적인 채팅 백업 도구는 원문 보존에 초점을 맞추지만, 대화를 분석 가능한 연구 문서로 바꾸지는 못한다.

Chat Paper AI는 비정형 대화를 학술 문서 구조로 변환한다. 결과물은 단순한 AI 답변이 아니라 제목, 초록, 서론, 연구 방법, 결과, 논의, 결론으로 구성된 논문형 초안이다.

## 2. 시스템 목표

- 카카오톡 대화 파일에 적합한 한국어 우선 업로드 경험 제공
- Markdown, text, JSON 기반 AI 대화 파일 지원
- 긴 대화 로그를 청크 단위로 요약한 뒤 최종 논문 생성
- 로그인 없이 사용할 수 있는 게스트 기반 흐름 유지
- 원본 파일 영구 저장을 피하고 파싱된 메시지를 익명화한 뒤 저장
- 결과를 연구 대시보드와 논문 리더 형태로 제공
- Vercel, PostgreSQL, Prisma, OpenAI 기반으로 배포 가능한 구조 유지

## 3. 전체 아키텍처

```mermaid
flowchart LR
  U[사용자] --> UI[Next.js App Router UI]
  UI --> Upload[/api/upload]
  Upload --> Parser[대화 파서]
  Parser --> Privacy[익명화 레이어]
  Privacy --> DB[(PostgreSQL via Prisma)]
  UI --> Analyze[/api/analyze]
  Analyze --> Lang[언어 감지]
  Lang --> PaperRecord[PROCESSING 논문 레코드]
  UI --> Generate[/api/papers/:paperId/generate]
  Generate --> Chunker[메시지 청킹]
  Chunker --> OpenAI[OpenAI 논문 생성 파이프라인]
  OpenAI --> DB
  DB --> Dashboard[연구 대시보드]
  DB --> Reader[학술 논문 리더]
  Reader --> Export[내보내기 액션]
```

프론트엔드와 백엔드는 Next.js App Router 애플리케이션 안에 함께 구성되어 있다. API Route는 업로드, 분석 시작, 논문 생성, 논문 조회를 담당한다. Prisma는 PostgreSQL 접근을 타입 안정적으로 처리하고, OpenAI 레이어는 청크 요약, 관계 분석, 논문 섹션 생성을 담당한다.

## 4. 업로드 및 파싱 흐름

업로드 API는 `.zip`, `.txt`, `.md`, `.json` 파일을 처리한다. 카카오톡 ZIP 내보내기 파일의 경우 서버는 ZIP 내부에서 가장 큰 `.txt` 파일을 찾아 주요 대화 본문으로 사용한다. ZIP이 아닌 파일은 텍스트로 직접 읽는다.

이후 시스템은 파일이 카카오톡 대화인지 AI 대화인지 판단한다. 카카오톡 로그는 발화자, 타임스탬프, 메시지 본문으로 정규화된다. AI 대화 파일은 JSON, Markdown, 일반 텍스트 형식에서 역할 기반 대화 구조로 변환된다.

파싱 이후에는 분석에 필요한 구조화 데이터만 저장한다. 원본 파일은 별도 다운로드 가능한 파일 객체로 보존하지 않는다.

## 5. 개인정보 보호와 게스트 세션

Chat Paper AI는 브라우저 범위의 게스트 세션 쿠키를 사용해 현재 방문자의 업로드와 논문을 연결한다. 이를 통해 로그인 없는 흐름을 유지하면서도 업로드, 결과, 논문 조회 경로에서 소유권 검사를 수행할 수 있다.

파싱된 메시지를 저장하기 전 개인정보 보호 레이어는 전화번호, 식별번호, 주소, 이름과 같은 일반적인 민감정보 패턴을 치환한다. 이 과정은 데이터베이스 저장 데이터와 AI 프롬프트로 전달되는 데이터의 노출 위험을 줄인다.

현재 개인정보 보호 모델은 MVP 속도와 사용성에 맞춘 경량 구조다. 운영 고도화 단계에서는 명시적 데이터 삭제, 보관 기간 정책, 감사 로그, 한국어 이름 및 주소에 대한 더 정교한 개체명 인식이 추가될 수 있다.

## 6. 데이터 모델

Prisma 스키마는 다섯 개의 핵심 엔티티를 중심으로 구성된다.

- `User`: 게스트 또는 인증 사용자 식별 정보
- `Upload`: 업로드 파일 메타데이터와 파싱 상태
- `ParsedMessage`: 정규화 및 익명화된 대화 메시지
- `Paper`: 생성된 논문 섹션과 분석 메타데이터
- `Export`: 내보내기 문서 메타데이터

관계는 삭제 전파를 고려해 설계되어 있다. 사용자는 업로드와 논문을 소유하고, 업로드는 파싱 메시지를 소유하며, 논문은 내보내기 기록을 소유한다. 이 구조는 향후 데이터 정리나 사용자 삭제 기능을 추가할 때 예측 가능한 정리 동작을 제공한다.

## 7. AI 논문 생성 파이프라인

논문 생성 파이프라인은 긴 대화 로그를 처리할 수 있도록 단계화되어 있다.

1. 파싱된 메시지를 시간순으로 불러온다.
2. AI 생성 전에 메시지를 다시 익명화한다.
3. 주요 언어를 감지하고 요청된 논문 언어를 저장한다.
4. 긴 대화를 모델 문맥과 호출 한도에 맞게 청크로 분할한다.
5. 각 청크에서 주제, 감정, 발화자 역학, 주요 이벤트를 요약한다.
6. 관계 분석을 통해 대화 유형, 문제 징후, 선택적 애정 점수를 추론한다.
7. 통합된 분석 컨텍스트를 기반으로 논문 섹션을 배치 단위로 생성한다.
8. 논문 상태를 `PROCESSING`에서 `COMPLETED` 또는 `FAILED`로 갱신한다.

OpenAI 호출에는 rate limit 응답에 대한 재시도 로직이 포함되어 있어 생성 과정의 안정성을 높인다.

## 8. 프론트엔드 경험

제품의 주요 화면은 네 가지로 구성된다.

- 랜딩 페이지: 간결한 제품 포지셔닝, 언어 전환, 테마 컨트롤
- 업로드 흐름: 드래그 앤 드롭 업로드, 단계별 로딩 상태, 명확한 CTA
- 결과 대시보드: 논문 메타데이터, 상태, 분석 프레이밍, 리더 진입
- 논문 리더: 제목, 초록, 연구 방법, 결과, 논의, 결론을 읽기 좋은 학술 문서 레이아웃으로 제공

UI는 재사용 가능한 레이아웃, 버튼, 카드, 배지, 카피 레이어 위에 구성되어 있다. 디자인 방향은 한국어 가독성, 프리미엄 SaaS 간격 체계, 다크모드 준비, 모바일 반응형 완성도에 초점을 둔다.

## 9. 배포 구조

플랫폼은 Vercel 배포를 기준으로 설계되어 있다.

- UI와 API Route를 위한 Next.js App Router
- 주 데이터베이스로 PostgreSQL 사용
- 설치 단계에서 Prisma Client 생성
- 환경변수 기반 OpenAI API Key 주입
- 업로드와 분석 API에 대한 Vercel 함수 실행 시간 설정

주요 환경변수는 다음과 같다.

```bash
OPENAI_API_KEY=
DATABASE_URL=
NEXTAUTH_SECRET=
NEXTAUTH_URL=
```

게스트 우선 흐름을 사용할 경우 OAuth 관련 환경변수는 선택적으로 둘 수 있다.

## 10. 안정성 고려사항

시스템은 다음과 같은 운영 안정성 장치를 포함한다.

- 파싱 전 파일 형식 검증
- 업로드 요청의 multipart form 검증
- 논문 조회 및 생성 전 소유권 검사
- 명시적인 논문 상태 전이
- OpenAI 생성 실패 시 `FAILED` 상태 저장
- 긴 생성 작업을 고려한 폴링 친화적 논문 조회
- OpenAI rate limit 오류에 대한 재시도 처리

향후 고도화 영역은 큐 기반 백그라운드 작업, 배포 환경별 업로드 용량 제한, 관측성 대시보드, 사용자 재시도 컨트롤, 자동 데이터 보관 기간 정책이다.

## 11. 기술적 의의

Chat Paper AI는 비정형 사용자 데이터를 구조화된 지식 산출물로 변환하는 풀스택 AI 제품 패턴을 보여준다. 이 프로젝트는 프론트엔드 제품 완성도, 파일 파싱, 데이터베이스 모델링, 개인정보 보호 전처리, AI 오케스트레이션, 문서형 UX를 하나의 배포 가능한 SaaS 애플리케이션으로 결합한다.

아키텍처는 파서, 논문 스타일, 내보내기 형식, 보관 정책, AI 모델을 독립적으로 확장할 수 있도록 모듈화되어 있다. 따라서 핵심 사용자 여정은 유지하면서도 제품 기능을 점진적으로 고도화할 수 있다.
